# 实验反思与总结

## 概述

本实验基于 Luo et al. (IEEE TITS, 2022) 的论文，复现了面向卡车-无人机协同路径规划的 HMOA（Hybrid Multi-Objective Optimization Approach）算法。该算法融合 NSGA-II 框架与 Pareto 局部搜索（PLS），针对带柔性时间窗的多目标优化问题（最小化运输成本 + 最大化客户满意度）进行了求解。通过在 20 客户和 50 客户实例上的测试，验证了算法有效性，并在复现过程中对初始化逻辑进行了 Bug 修复与性能优化（10.6× 加速）。

---

## 1. POMO、GA、OR 三类算法在目标值、运行效率上的优劣对比

### 1.1 POMO（基于学习的构造型方法）

| 维度 | 评价 |
|------|------|
| **目标值** | 在分布外泛化上较弱；对约束的满足依赖训练数据覆盖度。但在已知分布的小规模问题上可逼近最优解。 |
| **运行效率** | **推理极快**（ms 级输出完整解序列），但**训练成本极高**（需大量 GPU 资源和大规模算例）。 |
| **约束处理** | 硬约束需通过掩码（masking）嵌入 attention 机制，柔性约束（如软时间窗）较难精确建模。 |
| **适用场景** | 同分布大规模在线决策，或需秒级响应的场景。 |

### 1.2 GA / 进化算法（EA）

| 维度 | 评价 |
|------|------|
| **目标值** | 本实验中 HMOA（NSGA-II + PLS）在 20 客户实例上找到 30 个 Pareto 非支配解，最优成本 3,133.99，形成完整权衡曲线。多目标优化能力强于加权求和法。 |
| **运行效率** | **中等**（20 客户 13.89s，50 客户 37.68s）。随问题规模呈超线性增长，但可通过并行化缓解。 |
| **约束处理** | **灵活性最高**。修复启发式（Repair）可处理任意复杂约束（能耗、时间窗、同步约束等），不要求目标函数可导。本实验中的 Repair 策略（Algorithm 4）即为典型。 |
| **适用场景** | 学术研究与中小规模实际问题的多目标优化。 |

### 1.3 运筹优化 OR（精确算法 / 分支定界）

| 维度 | 评价 |
|------|------|
| **目标值** | **最优性保证**。可证明最优解或给出最优性间隙（optimality gap）。 |
| **运行效率** | **小规模极快**（数十节点），但随规模呈指数爆炸，实际中常限制在 ≤100 客户。 |
| **约束处理** | 需将约束建模为线性/混合整数规划（MILP）形式，建模成本高。本论文的约束(4)-(33) 即为完整的 MILP 建模，但仅作为问题定义，求解仍依靠元启发式。 |
| **适用场景** | 小规模标杆对比、最优解验证、以及问题下界估计。 |

### 1.4 三类算法的直观对比

| 维度 | POMO（学习型） | GA / EA（进化型） | OR（精确型） |
|------|:-------------:|:----------------:|:-----------:|
| 目标值质量 | ★★★☆（分布内） | ★★★★ | ★★★★★ |
| 求解速度 | ★★★★★ | ★★★☆ | ★★☆☆ |
| 约束灵活性 | ★★☆☆ | ★★★★★ | ★★★☆☆ |
| 可扩展性 | ★★★★☆ | ★★★★☆ | ★☆☆☆☆ |
| 实现复杂度 | ★★★★★ | ★★★☆☆ | ★★★★☆ |
| **综合推荐** | 在线/大规模/同分布 | **学术研究/多目标** | 小规模/最优性证明 |

> 对于本课题的 **EVRP-TW**，GA 类方法因其约束灵活性和多目标优化能力，是当前阶段的首选范式。

## 2. 在基线算法上新增能耗约束、时间窗约束时遇到的主要难点

### 2.1 能耗约束的难点

| 难点 | 具体表现 | 解决方案与思考 |
|------|---------|--------------|
| **非线性能耗模型** | 电池放电与载重、坡度、速度、温度均相关，简化为线性距离函数会损失精度。本论文使用 Euclidean 距离判定续航（约束 22），仅为一阶近似。 | 引入分段线性近似或基于物理模型的回归代理。 |
| **充电决策耦合** | 充电位置、时长、充电策略（满充/部分充）与路径规划强耦合，搜索空间爆炸。 | 采用两级决策：先路由后充电调度，或使用 Repair 修复充电可行性。 |
| **状态依赖续航** | 剩余电量随已行驶路径动态变化，非固定约束——这使得邻域搜索中"交换两个节点"后需重新评估整条路径的能耗可行性。 | 在 Repair 和邻域算子中加入增量式能耗更新，避免完全重新评估。 |

### 2.2 时间窗约束的难点

| 难点 | 具体表现 | 解决方案与思考 |
|------|---------|--------------|
| **硬时间窗 → 可行解稀少** | 在随机初始解中，满足硬时间窗的比例极低。本论文采用"柔性时间窗"（允许以满意度衰减偏离），避免了这一问题。 | 硬时间窗下需采用复杂的时间窗感知初始化（如 NearestNeighborTW）。 |
| **等待时间与同步** | 柔性时间窗下允许早到等待，但等待时间累积影响后续节点的时间可行性。本论文中通过仿真时间线计算到达时间（`_compute_arrival_times`）处理此问题。 | 需在评价函数中维护完整的时间线仿真，不能仅靠静态距离估算。 |
| **无人机与卡车的时间协同** | 无人机飞行到达节点后，卡车可能未到，产生等待约束（约束 25-26）。时间耦合加剧搜索难度。 | 分离无人机和卡车的到达时间，取 max 作为会合时间（约束 24）。 |

### 2.3 复合约束的交互效应

**"能耗 + 时间窗 + 多车协同"不是一个简单的约束叠加问题，而是约束间的非线性耦合带来了组合爆炸：**

```
能耗(time(Route)) ≤ 电池容量
time(node) ∈ [e_i, l_i]  (柔性)
launch_before_land:  ∀ drone flights
```

实验中观察到：当同时启用 3 架无人机时，仅车辆协同约束（同一位置不能同时发/收多机）就将候选飞行的空间压缩了 60% 以上。添加能耗约束后，部分无人机配送方案退化到卡车配送，降低了 Pareto 前沿的质量——这与论文中 Fig.6 的结论（增加无人机数量可改善双目标）高度一致。

## 3. 针对 EVRP-TW 模型构建的启发与思考

### 3.1 模型框架的借鉴

| 论文已有 | 对 EVRP-TW 的启发 |
|---------|-----------------|
| 5 部分染色体编码（卡车路由 + 无人机配送） | → 可扩展为 **EVRP-TW 染色体**：卡车路由 + 充电站选择 + 充电量决策 |
| Pareto 双目标优化（成本 + 满意度） | → 类似双目标：**成本 + 充电时间**，或 **成本 + 碳排放** |
| 柔性时间窗 (e_i, l_i) | → **硬时间窗 + 软时间窗混合模型**，允许超时但惩罚 |
| Repair 修复不可行节点 | → **ChargeRepair**：为电量不足的路径插入充电站 |
| 自适应 PLS 触发 | → **自适应局部搜索强度**：根据种群多样性动态调节 |

### 3.2 对 EVRP-TW 特有的关键改进方向

#### (1) 充电站编码

无人机/充电站与卡车的类比：论文中的"无人机"可类比为 EVRP 中的"充电站行程"。建议采用 **3 部分染色体**：

```
[卡车路由] | [充电站位置索引] | [充电量比例 (0-1)]
```

- 充电站索引指向卡车路由中的插入位置
- 充电量比例（continuous 0-1）决定充电至满电的百分比
- Repair 策略负责确保任何染色体都满足"续航 > 0"约束

#### (2) 能耗评价函数

论文的 `evaluate()` 仅跟踪卡车时间线。对 EVRP-TW，需实现 **分段能耗计算**：

```python
def evaluate_evrp(route):
    energy = battery_capacity
    for each_segment in route:
        energy -= consumption(segment_distance, load)
        if energy < 0:  # 需要充电
            insert_charging_station()
            energy = battery_capacity  # 满充
```

#### (3) 初始解构造

论文的 `NearestNeighborTW` 给出了很好的起点，但对 EVRP-TW 需扩展为 **E-Insertion Heuristic**：

```
while unvisited:
    # 选择最近且"在剩余续航内可达"的未访问节点
    # 若无可达节点，则插入最近的充电站
```

#### (4) 约束处理的优先级

基于本实验的经验，建议 EVRP-TW 的 Repair 按以下优先级处理约束：

```
1. 连通性（路径连续，起止于车场）
2. 时间窗（等待/延迟调整）
3. 能耗约束（插入充电站）
4. 容量约束（载重/电池）
```

该顺序确保**尽早修复最关键的拓扑约束**，避免在不可行解上浪费修复计算。

### 3.3 算法选择的建议

基于本实验对三类算法的对比分析，对 EVRP-TW 课题提出以下建议：

1. **主算法**：以 NSGA-II/III 为框架的**混合进化多目标算法**（如本实验的 HMOA），利用 Repair 策略处理复杂约束
2. **局部搜索**：集成 **E-Insertion**（针对能耗的专用邻域算子）和 **2-Opt\***（带时间窗约束的 2-Opt 变体）
3. **对标工具**：使用 OR-Tools（CP-SAT）求解小规模实例（≤20 客户），为进化算法提供最优性参考
4. **机器学习辅助**：若需处理大规模实例（≥200 客户），可考虑用 **AM（Attention Model）** 学习 Repair 策略中的"最佳充电站选择"决策

---

## 结论

本次 HMOA 复现实验表明：
1. **进化多目标算法**在解决带复杂约束的车辆路径问题时，在解质量和约束灵活性之间取得了最佳平衡，优于纯学习和纯运筹方法。
2. 约束间的**非线性耦合**（能耗 × 时间窗 × 路径拓扑）是 EVRP-TW 的核心挑战，需通过专用 Repair 和邻域算子在算法层面解耦。
3. 论文中 5 部分染色体的表示方法和自适应 PLS 触发机制可直接迁移至 EVRP-TW，只需将"无人机飞行"替换为"充电站行程"，将"续航约束"替换为"电池 SoC 约束"。
